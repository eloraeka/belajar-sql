Belajar syntax SQL

Urutan lengkap dari CREATE DATABASE sampai TRANSACTION, plus kamus fungsi string & tanggal · semua contoh bisa dijalankan.

mysql — belajar

01CREATE DATABASE

Membuat database baru. Nama database harus unik dan mengikuti aturan penamaan.

DDL
CREATE DATABASE sekolah_ku;
02USE DATABASE

Memilih database yang akan digunakan untuk operasi berikutnya.

DDL
USE sekolah_ku;
03CREATE TABLE

Membuat tabel baru beserta kolom-kolomnya.

DDL
CREATE TABLE siswa (
  id INT,
  nama VARCHAR(50),
  umur INT
);
04Data Type

Tipe data menentukan jenis nilai yang bisa disimpan di kolom.

DDL
CREATE TABLE contoh_tipe (
  id INT,
  nama VARCHAR(100),
  tanggal_lahir DATE,
  gaji DECIMAL(10,2),
  aktif BOOLEAN
);
05Constraint

Aturan validasi data: PRIMARY KEY, FOREIGN KEY, NOT NULL, UNIQUE, DEFAULT, CHECK.

DDL
CREATE TABLE mahasiswa (
  npm CHAR(10) PRIMARY KEY,
  nama VARCHAR(50) NOT NULL,
  jurusan VARCHAR(30) DEFAULT 'Informatika',
  usia INT CHECK (usia >= 17),
  email VARCHAR(100) UNIQUE
);
06REFERENCES

Menghubungkan foreign key ke primary key di tabel lain.

DDL
CREATE TABLE nilai (
  id INT PRIMARY KEY,
  npm CHAR(10),
  mata_kuliah VARCHAR(50),
  FOREIGN KEY (npm) REFERENCES mahasiswa(npm)
);
07ON DELETE

Aksi saat baris di tabel induk dihapus: CASCADE, SET NULL, RESTRICT.

DDL
CREATE TABLE nilai_ondelete (
  id INT PRIMARY KEY,
  npm CHAR(10),
  FOREIGN KEY (npm) REFERENCES mahasiswa(npm)
    ON DELETE CASCADE
);
08ON UPDATE CASCADE

Saat primary key di tabel induk diubah, foreign key di tabel anak ikut berubah.

DDL
CREATE TABLE nilai_onupdate (
  id INT PRIMARY KEY,
  npm CHAR(10),
  FOREIGN KEY (npm) REFERENCES mahasiswa(npm)
    ON UPDATE CASCADE
);
09ALTER TABLE

Mengubah struktur tabel yang sudah ada: menambah kolom, menambah constraint dengan nama alias, dan lainnya.

＋Tambah kolom baru

Kolom baru ditambahkan dengan ADD COLUMN. Nilai lama akan terisi NULL untuk kolom ini secara default.

DDL
ALTER TABLE mahasiswa
ADD COLUMN alamat TEXT;
✓Tambah kolom ENUM

Tipe ENUM membatasi nilai kolom hanya pada daftar pilihan yang ditentukan, contohnya jenis kelamin.

DDL
ALTER TABLE mahasiswa
ADD COLUMN jenis_kelamin ENUM('Male','Female');
🔒Tambah constraint dengan alias

Constraint tambahan bisa diberi nama sendiri (alias) memakai backtick, supaya lebih mudah dikenali atau dihapus di kemudian hari. Berikut memastikan format NPM dan batas usia minimum.

DDL
ALTER TABLE mahasiswa
ADD CONSTRAINT CHECK (npm REGEXP '^M[0-9]{3}$');

ALTER TABLE mahasiswa
ADD CONSTRAINT `Minimum usia` CHECK (usia >= 17);
10DROP TABLE

Menghapus tabel beserta seluruh data dan strukturnya.

DDL
DROP TABLE IF EXISTS nilai_ondelete;
11TRUNCATE TABLE

Menghapus semua data di tabel tapi struktur tetap ada.

DDL
TRUNCATE TABLE mahasiswa;
12INSERT

Menambah baris data ke dalam tabel.

DML
INSERT INTO mahasiswa (npm, nama, jurusan, usia, email)
VALUES ('M001', 'Budi', 'Informatika', 20, 'budi@mail.com');
13SELECT

Mengambil data dari tabel. Hierarki dasar sebuah query SELECT selalu berjalan dari FROM lalu baru disaring dengan WHERE.

▤Satu kolom
SELECT nama_kolom FROM table_name;
DML
SELECT nama FROM mahasiswa;
▦Banyak kolom
SELECT kolom1, kolom2 FROM table_name;
DML
SELECT nama, jurusan FROM mahasiswa;
✱Semua kolom
SELECT * FROM table_name;
DML
SELECT * FROM mahasiswa;
14WHERE · Operator Dasar

Menyaring baris berdasarkan syarat tertentu. Ada dua cara umum untuk membandingkan nilai: tanda = untuk kecocokan persis, dan LIKE untuk pencocokan pola (mendukung wildcard).

=Kecocokan nilai yang persis sama.
SELECT kolom FROM table WHERE kolom = kondisi;
DML
SELECT nama, jurusan, email
FROM mahasiswa
WHERE jurusan = 'Informatika';
LIKEPencocokan pola teks, bisa dipakai persis seperti = atau dengan wildcard.
SELECT kolom FROM table WHERE kolom LIKE kondisi;
DML
SELECT nama, jurusan, email
FROM mahasiswa
WHERE jurusan LIKE 'Informatika';
⌗Wildcard pada LIKE

Simbol % mengabaikan sejumlah karakter apa pun di posisinya, sedangkan REGEXP memakai simbol regex seperti ^ (awal string) dan $ (akhir string) untuk pola yang lebih presisi.

a%
Diawali huruf "a", sisanya bebas
%a
Diakhiri huruf "a"
%a%
Mengandung huruf "a" di mana saja
^A[0-9]{3}$
REGEXP: huruf A diikuti 3 digit angka
DML
SELECT nama FROM mahasiswa
WHERE nama LIKE 'B%';
> < >= <= !=Operator perbandingan untuk data angka.
DML
SELECT nama, usia
FROM mahasiswa
WHERE usia > 18;
15Kondisi Majemuk

Menggabungkan lebih dari satu syarat dalam satu WHERE menggunakan AND, OR, atau mempersingkat banyak OR dengan IN.

ANDKedua kondisi harus sama-sama terpenuhi.
DML
SELECT * FROM mahasiswa
WHERE jurusan = 'Informatika' AND usia > 18;
ORCukup salah satu kondisi yang terpenuhi.
DML
SELECT * FROM mahasiswa
WHERE jurusan = 'Informatika' OR jurusan = 'Sistem Informasi';
INMenyaring berdasarkan sekelompok nilai sekaligus — ringkasan dari banyak OR.
DML
SELECT * FROM mahasiswa
WHERE jurusan IN ('Informatika', 'Sistem Informasi', 'Teknik Komputer');
16Kamus String Function

Kumpulan fungsi bawaan SQL untuk mengolah data teks. Nama kolom hasil bisa diganti tampilannya memakai AS.

LEFT(kolom, n)Mengambil n karakter dari sisi kiri sebuah teks.
DML
SELECT LEFT(nama, 3) AS nama_singkat
FROM mahasiswa;
RIGHT(kolom, n)Mengambil n karakter dari sisi kanan sebuah teks.
DML
SELECT RIGHT(email, 8) AS domain
FROM mahasiswa;
UPPER / LOWER(kolom)Mengubah teks menjadi huruf kapital semua, atau huruf kecil semua.
DML
SELECT nama,
       LOWER(nama) AS nama_kecil,
       UPPER(nama) AS nama_besar
FROM mahasiswa;
CONCAT(...)Menggabungkan beberapa nilai kolom dan/atau teks statis menjadi satu string. Teks tambahan bisa diletakkan sebelum, sesudah, atau di kedua sisi data.
DML
SELECT CONCAT('Halo, ', nama, '! Selamat datang.') AS pesan
FROM mahasiswa;
LENGTH(kolom)Menghitung jumlah karakter dalam sebuah teks.
DML
SELECT nama, LENGTH(nama) AS panjang_nama
FROM mahasiswa;
REPLACE(kolom, cari, ganti)Mengganti setiap kemunculan sebuah kata dalam teks dengan kata lain.
DML
SELECT REPLACE(jurusan, 'Informatika', 'Ilmu Komputer') AS jurusan_baru
FROM mahasiswa;
SUBSTR(kolom, start, n)Mengambil potongan teks mulai dari indeks tertentu sepanjang n karakter.
DML
SELECT SUBSTR(nama, 2, 3) AS potongan
FROM mahasiswa;
17Kamus Date Function

Fungsi bawaan untuk membaca dan menghitung selisih tanggal / waktu.

CURDATE()Menampilkan tanggal hari ini (tanpa jam).
DML
SELECT CURDATE() AS hari_ini;
NOW()Menampilkan tanggal sekaligus jam saat ini (real time), berbeda dengan CURDATE yang hanya tanggal.
DML
SELECT NOW() AS waktu_sekarang;
DATEDIFF(a, b)Menghitung selisih dua tanggal dalam satuan hari, hasilnya a − b.
DML
SELECT DATEDIFF(CURDATE(), tanggal_lahir) AS selisih_hari
FROM contoh_tipe;
TIMESTAMPDIFF(unit, a, b)Sama seperti DATEDIFF tapi bisa memilih satuan (DAY, MONTH, YEAR) dan hasilnya b − a.
DML
SELECT TIMESTAMPDIFF(MONTH, tanggal_lahir, CURDATE()) AS umur_dalam_bulan
FROM contoh_tipe;
DAY / MONTH / YEAR(kolom)Mengambil bagian hari, bulan, atau tahun saja dari sebuah tanggal.
DML
SELECT DAY(tanggal_lahir) AS tgl,
       MONTH(tanggal_lahir) AS bln,
       YEAR(tanggal_lahir) AS thn
FROM contoh_tipe;
18ORDER BY

Mengurutkan hasil query. ASC untuk menaik (default), DESC untuk menurun.

DML
SELECT * FROM mahasiswa
ORDER BY usia DESC;
19Aggregate Function

COUNT, SUM, AVG, MAX, MIN — meringkas banyak baris menjadi satu nilai.

DML
SELECT COUNT(*) AS total,
       AVG(usia) AS rata_usia
FROM mahasiswa;
20GROUP BY

Mengelompokkan data untuk agregasi per kelompok.

DML
SELECT jurusan, COUNT(*) AS jumlah
FROM mahasiswa
GROUP BY jurusan;
21HAVING

Menyaring hasil GROUP BY — mirip WHERE, tapi bekerja setelah data dikelompokkan/diagregasi.

DML
SELECT jurusan, COUNT(*) AS jumlah
FROM mahasiswa
GROUP BY jurusan
HAVING COUNT(*) > 0;
22Simple Join

Cara lama menghubungkan dua tabel tanpa kata kunci JOIN: cukup sebutkan kedua tabel di FROM dipisah koma, lalu hubungkan atributnya di WHERE. Tidak harus foreign key ke primary key — dua kolom apa pun yang nilainya sama bisa dipakai.

⚠Tanpa WHERE → cartesian product

Jika dua tabel hanya digabung dengan koma tanpa WHERE, setiap baris tabel pertama dipasangkan dengan semua baris tabel kedua (permutasi penuh) — biasanya bukan yang diinginkan.

⚠
Hati-hati: jumlah baris hasilnya = jumlah baris tabel A × jumlah baris tabel B.
DML
SELECT *
FROM missions, astronauts;
🔗Dengan WHERE → menyamakan relasi

Menambahkan WHERE untuk menyamakan kolom penghubung membuat hasilnya sinkron, setara dengan INNER JOIN.

DML
SELECT missionId, missions.astronautId, astronautName
FROM missions, astronauts
WHERE missions.astronautId = astronauts.astronautId;
23INNER JOIN

Sintaks modern untuk menggabungkan tabel berdasarkan relasi — lebih jelas dibaca dibanding Simple Join.

DML
SELECT mahasiswa.nama, nilai.mata_kuliah
FROM mahasiswa
INNER JOIN nilai ON mahasiswa.npm = nilai.npm;
24UPDATE

Mengubah nilai pada baris yang sudah ada. Bisa mengubah satu kolom atau beberapa kolom sekaligus, dipisah koma.

DML
UPDATE mahasiswa
SET usia = 21
WHERE npm = 'M001';
DML
UPDATE mahasiswa
SET nama = 'Budi Santoso', usia = 22
WHERE npm = 'M001';
25DELETE

Menghapus baris berdasarkan kondisi — tanpa SELECT apa pun, langsung dari nama tabelnya.

DML
DELETE FROM mahasiswa
WHERE npm = 'M001';
⚠
Ingat: selalu sertakan WHERE, kalau tidak seluruh baris di tabel akan terhapus.
26TRANSACTION

Membungkus beberapa query menjadi satu sesi kerja (unit kerja) yang bisa dibatalkan seluruhnya jika salah satu langkah gagal, atau disimpan permanen jika semua sukses.

🧾Contoh alur transaksi
DML
START TRANSACTION;

UPDATE missions
SET missionStatus = 'Completed'
WHERE missionId = 'MI0002';

UPDATE missionlogs
SET logMessage = 'Something error on this event'
WHERE missionId = 'MI0002';

ROLLBACK;
⚖ROLLBACK vs COMMIT

Selama transaksi belum diakhiri, semua perubahan sifatnya sementara (temporary) dan hanya terlihat dalam sesi itu.

ROLLBACK

Membatalkan seluruh perubahan yang dilakukan sejak START TRANSACTION, mengembalikan data seperti semula. Dipakai saat salah satu langkah gagal atau tidak sesuai harapan.

COMMIT

Menyimpan seluruh perubahan secara permanen ke database. Setelah COMMIT, perubahan tidak bisa lagi dibatalkan dengan ROLLBACK.
