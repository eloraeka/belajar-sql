01SELECT & Memilih Kolom

Perintah paling dasar untuk mengambil data dari sebuah tabel. Bisa memilih satu kolom, beberapa kolom, atau seluruh kolom.

▸ Satu atau beberapa kolom

Untuk mengambil satu kolom, tulis nama kolomnya setelah SELECT. Untuk beberapa kolom, pisahkan dengan koma. Urutan kolom pada hasil akan mengikuti urutan penulisan di query, bukan urutan aslinya di tabel.

mscustomer
SELECT customerName, customerAddress, customerGender
FROM mscustomer;
▸ Semua kolom dengan *

Tanda bintang * berarti "semua kolom". Praktis untuk eksplorasi awal, tapi di aplikasi nyata sebaiknya sebutkan nama kolom secara eksplisit — lebih cepat dan lebih jelas kolom apa saja yang sebenarnya dipakai.

mscustomer
SELECT * FROM mscustomer;
02WHERE & Conditional Query

Menyaring baris data berdasarkan syarat tertentu. Urutan eksekusi logisnya: FROM dulu (menentukan tabel sumber), baru WHERE (menyaring barisnya).

▸ Kesetaraan dengan =

Operator = mencocokkan nilai secara persis (exact match).

mscustomer
SELECT customerName, customerAddress, customerGender
FROM mscustomer
WHERE customerGender = 'female';
▸ Multiple condition: AND & OR

AND mengharuskan kedua syarat terpenuhi sekaligus. OR cukup salah satu syarat saja yang terpenuhi.

msproduct — AND
SELECT * FROM msproduct
WHERE categoryId = 'CT002' AND productStock = 100;
msproduct — OR
SELECT * FROM msproduct
WHERE categoryId = 'CT002' OR categoryId = 'CT003';
▸ IN sebagai group condition

IN (...) adalah cara singkat untuk menulis banyak kondisi OR pada kolom yang sama.

msproduct
SELECT * FROM msproduct
WHERE categoryId IN ('CT002', 'CT003', 'CT005');
▸ Operator pembanding angka

Untuk kolom bertipe angka, bisa memakai operator pembanding seperti >, <, >=, <=.

msproduct
SELECT * FROM msproduct
WHERE productPrice > 200000;
03Wildcard, LIKE & REGEXP

Wildcard dipakai bersama LIKE untuk mencocokkan sebagian teks, bukan teks yang persis sama. Untuk pola yang lebih rumit, gunakan REGEXP.

▸ % sebagai penampung karakter bebas

Tanda % berarti "abaikan berapa pun karakter di posisi ini". Contoh 'a%' berarti mencari data yang diawali huruf a, tidak peduli apa pun setelahnya.

mscustomer
SELECT * FROM mscustomer
WHERE customerName LIKE 'a%';
▸ _ untuk tepat satu karakter

Setiap _ di dalam pola mewakili persis satu karakter, apa pun karakternya. Kalau % itu "sembarang panjang", _ itu "sembarang isi, tapi panjangnya tetap 1".

mscustomer
SELECT * FROM mscustomer
WHERE customerName LIKE 'A__ Wijaya';
▸ REGEXP untuk pola yang lebih fleksibel

REGEXP memakai regular expression, jauh lebih powerful dari LIKE. Beberapa simbol yang sering dipakai: ^ = awal teks, $ = akhir teks, [0-9] = salah satu digit, {n} = tepat n kali pengulangan, | = "atau".

astronauts — format id
SELECT * FROM astronauts
WHERE astronautId REGEXP '^AS[0-9]{3}$';
mscustomer — awalan A atau B
SELECT * FROM mscustomer
WHERE customerName REGEXP '^(A|B)';
04String Function

Fungsi bawaan SQL untuk mengolah data bertipe teks langsung di dalam query.

▸ LEFT & RIGHT

LEFT(kolom, n) mengambil n karakter dari sisi kiri. RIGHT(kolom, n) mengambil n karakter dari sisi kanan. Alias AS dipakai untuk mengganti nama kolom pada hasil tampilan.

mscustomer
SELECT LEFT(customerName, 5) AS customerName
FROM mscustomer;
▸ LOWER & UPPER

Mengubah teks menjadi huruf kecil semua (LOWER) atau huruf besar semua (UPPER).

mscustomer
SELECT customerName,
       LOWER(customerName) AS lowerCN,
       UPPER(customerName) AS upperCN
FROM mscustomer;
▸ CONCAT

Menggabungkan beberapa nilai (teks tetap dan/atau nilai kolom) menjadi satu string. Bisa taruh teks tambahan di depan, belakang, atau di kedua sisi.

mscustomer
SELECT CONCAT('Customer ', customerName, ' Thank you') AS customerMessage
FROM mscustomer;
▸ LENGTH, REPLACE, SUBSTR

LENGTH(kolom) menghitung jumlah karakter. REPLACE(kolom, dicari, pengganti) mengganti sebagian teks. SUBSTR(kolom, start, panjang) mengambil potongan teks mulai dari indeks tertentu.

msproduct
SELECT productName,
       REPLACE(productName, 'Salmon', 'Tuna') AS renamed,
       SUBSTR(productName, 2, 5) AS potongan
FROM msproduct;
▸ TRIM & ROUND

TRIM(kolom) membuang spasi kosong di awal dan akhir teks — berguna untuk data yang tidak rapi. ROUND(angka, n) membulatkan angka desimal ke n digit di belakang koma.

msproduct
SELECT TRIM('  Keyboard Mekanik  ') AS rapi,
       ROUND(productPrice * 1.11, 2) AS hargaPlusPajak
FROM msproduct;
05Date Function

Fungsi untuk membaca dan menghitung selisih tanggal.

▸ CURDATE() & NOW()

CURDATE() mengembalikan tanggal hari ini saja. NOW() mengembalikan tanggal sekaligus jam saat query dijalankan.

function
SELECT CURDATE() AS tanggalHariIni, NOW() AS waktuSekarang;
▸ DATEDIFF & TIMESTAMPDIFF

DATEDIFF(a, b) menghitung selisih dua tanggal dalam satuan hari (hasilnya a − b). TIMESTAMPDIFF(unit, a, b) lebih fleksibel karena satuannya bisa ditentukan sendiri (MONTH, YEAR, dst), dan hasilnya b − a — jadi urutan parameternya kebalik dari DATEDIFF, perlu hati-hati.

transactionheader
SELECT transactionDate,
       DATEDIFF(curdate(), transactionDate) AS selisihHari,
       TIMESTAMPDIFF(MONTH, transactionDate, curdate()) AS selisihBulan
FROM transactionheader;
▸ DAY, MONTH, YEAR

Mengambil bagian tanggal (hari), bulan, atau tahun dari sebuah kolom tanggal.

transactionheader
SELECT transactionDate,
       DAY(transactionDate)   AS tgl,
       MONTH(transactionDate) AS bln,
       YEAR(transactionDate)  AS thn
FROM transactionheader;
06ORDER BY, LIMIT & DISTINCT

Mengatur urutan tampil hasil, membatasi jumlah baris, dan membuang nilai duplikat.

▸ ORDER BY — mengurutkan hasil

ORDER BY kolom ASC mengurutkan naik (default kalau tidak ditulis), ORDER BY kolom DESC mengurutkan turun. Bisa juga mengurutkan berdasarkan lebih dari satu kolom, dipisah koma.

msproduct
SELECT productName, productPrice
FROM msproduct
ORDER BY productPrice DESC;
▸ LIMIT — membatasi jumlah baris

LIMIT n membatasi hasil hanya n baris pertama. Sering dipakai bersama ORDER BY, misalnya untuk mencari "3 produk termurah".

msproduct
SELECT productName, productPrice
FROM msproduct
ORDER BY productPrice ASC
LIMIT 3;
▸ DISTINCT — membuang duplikat

DISTINCT hanya menampilkan nilai yang unik, kalau ada baris dengan nilai yang sama persis akan ditampilkan sekali saja.

msproduct
SELECT DISTINCT categoryId
FROM msproduct;
07Aggregate Function & GROUP BY

Fungsi agregat meringkas banyak baris menjadi satu nilai (total, rata-rata, dsb). Kalau digabung dengan GROUP BY, peringkasan dilakukan per kelompok.

▸ COUNT, SUM, AVG, MAX, MIN

COUNT(*) menghitung jumlah baris. SUM menjumlahkan, AVG merata-ratakan, MAX/MIN mencari nilai tertinggi/terendah pada sebuah kolom angka.

msproduct
SELECT COUNT(*) AS totalProduk,
       MAX(productPrice) AS termahal,
       MIN(productPrice) AS termurah
FROM msproduct;
▸ GROUP BY — meringkas per kelompok

Kolom yang ada di GROUP BY menjadi dasar pengelompokan; setiap kelompok diringkas jadi satu baris hasil lewat fungsi agregat.

msproduct
SELECT categoryId,
       COUNT(*) AS jumlah,
       AVG(productPrice) AS rataRata
FROM msproduct
GROUP BY categoryId;
▸ HAVING — menyaring hasil GROUP BY

WHERE menyaring baris sebelum dikelompokkan, sedangkan HAVING menyaring kelompok setelah agregat dihitung. Karena itu HAVING bisa memakai hasil fungsi agregat, sementara WHERE tidak.

msproduct
SELECT categoryId, SUM(productStock) AS totalStok
FROM msproduct
GROUP BY categoryId
HAVING SUM(productStock) > 100;
08BETWEEN & IS NULL

Dua kondisi WHERE tambahan yang sering dipakai: rentang nilai, dan pengecekan data kosong.

▸ BETWEEN — rentang nilai

kolom BETWEEN a AND b setara dengan kolom >= a AND kolom <= b — batas atas dan bawah ikut termasuk (inclusive).

msproduct
SELECT * FROM msproduct
WHERE productPrice BETWEEN 100000 AND 500000;
▸ IS NULL & IS NOT NULL

Nilai kosong (NULL) tidak bisa dicek dengan = NULL karena NULL berarti "tidak diketahui", bukan sebuah nilai. Harus memakai IS NULL atau IS NOT NULL.

astronauts
SELECT * FROM astronauts
WHERE gender IS NULL;
astronauts
SELECT * FROM astronauts
WHERE gender IS NOT NULL;
09Simple Join

Menggabungkan data dari dua tabel lewat atribut yang nilainya sama, tanpa memakai keyword JOIN. Kolom yang disamakan tidak harus foreign key ke primary key, cukup dua atribut yang nilainya berkorespondensi.

▸ Tanpa WHERE → cartesian product

Menulis dua nama tabel di FROM tanpa syarat penghubung akan menghasilkan semua kemungkinan pasangan baris (cartesian product / permutasi), bukan data yang benar-benar berelasi.

missions × astronauts
SELECT * FROM missions, astronauts;
▸ Dengan WHERE → data yang berelasi

Menambahkan WHERE tabelA.kolom = tabelB.kolom menyaring hanya pasangan baris yang memang saling berhubungan.

missions ⋈ astronauts
SELECT missionId, missions.astronautId, astronautName
FROM missions, astronauts
WHERE missions.astronautId = astronauts.astronautId;
10INNER JOIN & LEFT JOIN

Cara standar menggabungkan tabel memakai keyword JOIN ... ON .... Lebih jelas dibaca dibanding menaruh dua tabel di FROM lalu menyaring di WHERE, dan hasilnya sama untuk INNER JOIN.

▸ INNER JOIN — hanya baris yang cocok

INNER JOIN ... ON syarat hanya mengembalikan baris yang punya pasangan di kedua tabel. Sama seperti simple join di atas, tapi lebih mudah dibaca karena syarat penghubung ditulis dekat nama tabel yang digabung.

missions INNER JOIN astronauts
SELECT missionId, astronauts.astronautId, astronautName
FROM missions
INNER JOIN astronauts ON missions.astronautId = astronauts.astronautId;
▸ LEFT JOIN — semua baris tabel kiri tetap tampil

LEFT JOIN tetap menampilkan semua baris dari tabel di sisi kiri, meskipun tidak ada pasangannya di tabel kanan — kolom dari tabel kanan akan berisi NULL untuk baris yang tidak berpasangan.

missions LEFT JOIN astronauts
SELECT missionId, astronautName
FROM missions
LEFT JOIN astronauts ON missions.astronautId = astronauts.astronautId;
11INSERT INTO — Menambah Baris

Bagian dari DML, dipakai untuk menambah baris data baru ke sebuah tabel.

▸ Menyebutkan nama kolom secara eksplisit

Cara paling aman: sebutkan nama kolom yang mau diisi, lalu nilainya di VALUES sesuai urutan. Kolom yang tidak disebut akan diisi NULL atau nilai default.

astronauts
INSERT INTO astronauts (astronautId, astronautName, experienceYears)
VALUES ('AS004', 'Rudi Hartono', 5);
▸ Tanpa menyebut nama kolom

Kalau nama kolom tidak disebutkan, nilai di VALUES harus lengkap dan urut persis sesuai struktur tabel — cara ini lebih rawan salah kalau struktur tabel berubah.

astronauts
INSERT INTO astronauts
VALUES ('AS005', 'Nina Putri', 7, 'Female');
⚠️
Kalau jumlah atau urutan nilai di VALUES tidak cocok dengan jumlah/urutan kolom di tabel, query akan error atau data masuk ke kolom yang salah. Menyebutkan nama kolom secara eksplisit jauh lebih aman.
12UPDATE — Mengubah Isi Baris

Bagian dari DML (Data Manipulation Language), dipakai untuk mengubah nilai pada baris yang sudah ada.

▸ Update satu atau beberapa kolom sekaligus

Selalu sertakan WHERE agar perubahan hanya menyasar baris yang dimaksud — tanpa WHERE, seluruh baris di tabel akan ikut berubah.

astronauts
UPDATE astronauts
SET experienceYears = 10
WHERE astronautName = 'Maya Chen';

UPDATE astronauts
SET astronautName = 'Maya C.', experienceYears = 20
WHERE astronautId = 'AS002';
13ALTER TABLE

Bagian dari DDL (Data Definition Language) — mengubah struktur tabel: menambah kolom, menambah aturan (constraint), atau mengganti nama.

▸ Menambah kolom baru
astronauts
ALTER TABLE astronauts
ADD COLUMN gender ENUM('Male','Female');
▸ Menambah constraint (aturan validasi)

Constraint membatasi nilai apa saja yang boleh masuk ke sebuah kolom. Memberi nama pada constraint (lewat CONSTRAINT nama_constraint) membuatnya lebih mudah dikenali dan dihapus nanti.

astronauts
ALTER TABLE astronauts
ADD CONSTRAINT chk_id_format CHECK (astronautId REGEXP '^AS[0-9]{3}$');

ALTER TABLE astronauts
ADD CONSTRAINT chk_min_experience CHECK (experienceYears >= 3);
🔧
nama constraint ditulis sebelum CHECK, yaitu ADD CONSTRAINT nama_constraint CHECK(...). Nama constraint juga sebaiknya tanpa spasi (pakai underscore) supaya tidak perlu backtick.
14DELETE — Menghapus Baris

Menghapus seluruh baris yang cocok dengan syarat WHERE — bukan menghapus kolom tertentu saja.

missionlogs
DELETE FROM missionlogs
WHERE missionLogId = 'ML001';
⚠️
Hati-hati: DELETE FROM nama_tabel; tanpa WHERE akan menghapus seluruh baris di tabel tersebut. Selalu cek dulu dengan SELECT memakai syarat yang sama sebelum menjalankan DELETE.
15TRANSACTION

Membungkus beberapa query menjadi satu sesi perubahan. Kalau di tengah jalan ada yang gagal atau belum yakin, seluruh perubahan sementara ini bisa dibatalkan sekaligus.

▸ Alur START TRANSACTION → ROLLBACK
missions + missionlogs
START TRANSACTION;

UPDATE missions
SET missionStatus = 'Completed'
WHERE missionId = 'MI002';

UPDATE missionlogs
SET logMessage = 'Something error in this event'
WHERE missionId = 'MI002';

SELECT missions.missionId, missionStatus, logMessage
FROM missions, missionlogs
WHERE missions.missionId = missionlogs.missionLogId;

ROLLBACK;
▸ ROLLBACK vs COMMIT

ROLLBACK membatalkan semua perubahan yang dilakukan sejak START TRANSACTION, mengembalikan data ke kondisi semula. COMMIT sebaliknya — mengunci perubahan tersebut agar permanen tersimpan di database.
