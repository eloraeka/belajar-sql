01SELECT & Memilih Kolom

Perintah paling dasar untuk mengambil data dari sebuah tabel. Bisa memilih satu kolom, beberapa kolom, atau seluruh kolom.

▸ Satu atau beberapa kolom

Untuk mengambil satu kolom, tulis nama kolomnya setelah SELECT. Untuk beberapa kolom, pisahkan dengan koma. Urutan kolom pada hasil akan mengikuti urutan penulisan di query, bukan urutan aslinya di tabel.

mscustomer
SELECT customerName, customerAddress, customerGender
FROM mscustomer;
▸ Semua kolom dengan *

Tanda bintang * berarti "semua kolom". Praktis untuk eksplorasi awal, tapi di aplikasi nyata sebaiknya sebutkan nama kolom secara eksplisit — lebih cepat dan lebih jelas kolom apa saja yang sebenarnya dipakai.

mscustomer
SELECT * FROM mscustomer;
02WHERE & Conditional Query

Menyaring baris data berdasarkan syarat tertentu. Urutan eksekusi logisnya: FROM dulu (menentukan tabel sumber), baru WHERE (menyaring barisnya).

▸ Kesetaraan dengan =

Operator = mencocokkan nilai secara persis (exact match).

mscustomer
SELECT customerName, customerAddress, customerGender
FROM mscustomer
WHERE customerGender = 'female';
▸ Multiple condition: AND & OR

AND mengharuskan kedua syarat terpenuhi sekaligus. OR cukup salah satu syarat saja yang terpenuhi.

msproduct — AND
SELECT * FROM msproduct
WHERE categoryId = 'CT002' AND productStock = 100;
msproduct — OR
SELECT * FROM msproduct
WHERE categoryId = 'CT002' OR categoryId = 'CT003';
▸ IN sebagai group condition

IN (...) adalah cara singkat untuk menulis banyak kondisi OR pada kolom yang sama.

msproduct
SELECT * FROM msproduct
WHERE categoryId IN ('CT002', 'CT003', 'CT005');
▸ Operator pembanding angka

Untuk kolom bertipe angka, bisa memakai operator pembanding seperti >, <, >=, <=.

msproduct
SELECT * FROM msproduct
WHERE productPrice > 200000;
🔧
Koreksi: di catatan aslinya ada contoh customerGender LIKE 'female' tanpa tanda %. Ini valid, tapi secara hasil sama persis dengan customerGender = 'female' — LIKE baru terasa bedanya kalau dipakai bersama wildcard (% atau _). Jadi untuk pencocokan persis, lebih jelas pakai =; pakai LIKE saat memang butuh pencocokan sebagian.
03Wildcard & LIKE

Wildcard dipakai bersama LIKE untuk mencocokkan sebagian teks, bukan teks yang persis sama.

▸ % sebagai penampung karakter bebas

Tanda % berarti "abaikan berapa pun karakter di posisi ini". Contoh 'a%' berarti mencari data yang diawali huruf a, tidak peduli apa pun setelahnya.

mscustomer
SELECT * FROM mscustomer
WHERE customerName LIKE 'a%';
💡
Selain %, ada juga _ yang mewakili tepat satu karakter bebas (misalnya 'A_i' cocok dengan "Adi" atau "Ani"). Simbol ^ dan $ yang disebut di catatan aslinya adalah bagian dari regular expression (dipakai lewat REGEXP), bukan bagian dari LIKE — ^ menandai awal teks dan $ menandai akhir teks.
04String Function

Fungsi bawaan SQL untuk mengolah data bertipe teks langsung di dalam query.

▸ LEFT & RIGHT

LEFT(kolom, n) mengambil n karakter dari sisi kiri. RIGHT(kolom, n) mengambil n karakter dari sisi kanan. Alias AS dipakai untuk mengganti nama kolom pada hasil tampilan.

mscustomer
SELECT LEFT(customerName, 5) AS customerName
FROM mscustomer;
▸ LOWER & UPPER

Mengubah teks menjadi huruf kecil semua (LOWER) atau huruf besar semua (UPPER).

mscustomer
SELECT customerName,
       LOWER(customerName) AS lowerCN,
       UPPER(customerName) AS upperCN
FROM mscustomer;
▸ CONCAT

Menggabungkan beberapa nilai (teks tetap dan/atau nilai kolom) menjadi satu string. Bisa taruh teks tambahan di depan, belakang, atau di kedua sisi.

mscustomer
SELECT CONCAT('Customer ', customerName, ' Thank you') AS customerMessage
FROM mscustomer;
▸ LENGTH, REPLACE, SUBSTR

LENGTH(kolom) menghitung jumlah karakter. REPLACE(kolom, dicari, pengganti) mengganti sebagian teks. SUBSTR(kolom, start, panjang) mengambil potongan teks mulai dari indeks tertentu.

msproduct
SELECT productName,
       REPLACE(productName, 'Salmon', 'Tuna') AS renamed,
       SUBSTR(productName, 2, 5) AS potongan
FROM msproduct;
05Date Function

Fungsi untuk membaca dan menghitung selisih tanggal.

▸ CURDATE() & NOW()

CURDATE() mengembalikan tanggal hari ini saja. NOW() mengembalikan tanggal sekaligus jam saat query dijalankan.

function
SELECT CURDATE() AS tanggalHariIni, NOW() AS waktuSekarang;
▸ DATEDIFF & TIMESTAMPDIFF

DATEDIFF(a, b) menghitung selisih dua tanggal dalam satuan hari (hasilnya a − b). TIMESTAMPDIFF(unit, a, b) lebih fleksibel karena satuannya bisa ditentukan sendiri (MONTH, YEAR, dst), dan hasilnya b − a — jadi urutan parameternya kebalik dari DATEDIFF, perlu hati-hati.

transactionheader
SELECT transactionDate,
       DATEDIFF(curdate(), transactionDate) AS selisihHari,
       TIMESTAMPDIFF(MONTH, transactionDate, curdate()) AS selisihBulan
FROM transactionheader;
▸ DAY, MONTH, YEAR

Mengambil bagian tanggal (hari), bulan, atau tahun dari sebuah kolom tanggal.

transactionheader
SELECT transactionDate,
       DAY(transactionDate)   AS tgl,
       MONTH(transactionDate) AS bln,
       YEAR(transactionDate)  AS thn
FROM transactionheader;
06Simple Join

Menggabungkan data dari dua tabel lewat atribut yang nilainya sama, tanpa memakai keyword JOIN. Kolom yang disamakan tidak harus foreign key ke primary key, cukup dua atribut yang nilainya berkorespondensi.

▸ Tanpa WHERE → cartesian product

Menulis dua nama tabel di FROM tanpa syarat penghubung akan menghasilkan semua kemungkinan pasangan baris (cartesian product / permutasi), bukan data yang benar-benar berelasi.

missions × astronauts
SELECT * FROM missions, astronauts;
▸ Dengan WHERE → data yang berelasi

Menambahkan WHERE tabelA.kolom = tabelB.kolom menyaring hanya pasangan baris yang memang saling berhubungan.

missions ⋈ astronauts
SELECT missionId, missions.astronautId, astronautName
FROM missions, astronauts
WHERE missions.astronautId = astronauts.astronautId;
07UPDATE — Mengubah Isi Baris

Bagian dari DML (Data Manipulation Language), dipakai untuk mengubah nilai pada baris yang sudah ada.

▸ Update satu atau beberapa kolom sekaligus

Selalu sertakan WHERE agar perubahan hanya menyasar baris yang dimaksud — tanpa WHERE, seluruh baris di tabel akan ikut berubah.

astronauts
UPDATE astronauts
SET experienceYears = 10
WHERE astronautName = 'Maya Chen';

UPDATE astronauts
SET astronautName = 'Maya C.', experienceYears = 20
WHERE astronautId = 'AS002';
08ALTER TABLE

Bagian dari DDL (Data Definition Language) — mengubah struktur tabel: menambah kolom, menambah aturan (constraint), atau mengganti nama.

▸ Menambah kolom baru
astronauts
ALTER TABLE astronauts
ADD COLUMN gender ENUM('Male','Female');
▸ Menambah constraint (aturan validasi)

Constraint membatasi nilai apa saja yang boleh masuk ke sebuah kolom. Memberi nama pada constraint (lewat CONSTRAINT nama_constraint) membuatnya lebih mudah dikenali dan dihapus nanti.

astronauts
ALTER TABLE astronauts
ADD CONSTRAINT chk_id_format CHECK (astronautId REGEXP '^AS[0-9]{3}$');

ALTER TABLE astronauts
ADD CONSTRAINT chk_min_experience CHECK (experienceYears >= 3);
🔧
Koreksi: di catatan aslinya, nama constraint ditulis setelah CHECK(...) memakai backtick, misalnya ADD CONSTRAINT `Minimum experience years` CHECK(...). Urutan standarnya terbalik: nama constraint ditulis sebelum CHECK, yaitu ADD CONSTRAINT nama_constraint CHECK(...). Nama constraint juga sebaiknya tanpa spasi (pakai underscore) supaya tidak perlu backtick.
09DELETE — Menghapus Baris

Menghapus seluruh baris yang cocok dengan syarat WHERE — bukan menghapus kolom tertentu saja.

missionlogs
DELETE FROM missionlogs
WHERE missionLogId = 'ML001';
⚠️
Hati-hati: DELETE FROM nama_tabel; tanpa WHERE akan menghapus seluruh baris di tabel tersebut. Selalu cek dulu dengan SELECT memakai syarat yang sama sebelum menjalankan DELETE.
10TRANSACTION

Membungkus beberapa query menjadi satu sesi perubahan. Kalau di tengah jalan ada yang gagal atau belum yakin, seluruh perubahan sementara ini bisa dibatalkan sekaligus.

▸ Alur START TRANSACTION → ROLLBACK
missions + missionlogs
START TRANSACTION;

UPDATE missions
SET missionStatus = 'Completed'
WHERE missionId = 'MI002';

UPDATE missionlogs
SET logMessage = 'Something error in this event'
WHERE missionId = 'MI002';

SELECT missions.missionId, missionStatus, logMessage
FROM missions, missionlogs
WHERE missions.missionId = missionlogs.missionLogId;

ROLLBACK;
▸ ROLLBACK vs COMMIT

ROLLBACK membatalkan semua perubahan yang dilakukan sejak START TRANSACTION, mengembalikan data ke kondisi semula. COMMIT sebaliknya — mengunci perubahan tersebut agar permanen tersimpan di database.
